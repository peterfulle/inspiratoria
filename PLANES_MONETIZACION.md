## 🎯 ESTRATEGIA DE PRICING

### 1. Ancla Psicológica (Decoy Pricing)

```
TÁCTICA: Hacer Growth parecer mejor deal

Starter:    $3,200/año  = $160/usuario  (20 users)
Growth:     $9,600/año  = $120/usuario  (80 users)  ← MEJOR VALUE
Enterprise: $58,000/año = $193/usuario  (300 users)

🎯 Objetivo: 60% clientes eligen Growth (sweet spot)
```

### 2. Feature Gating Estratégico

**Starter → Growth (triggers de upgrade):**
- ❌ Solo 1 programa → Si quieren más programas
- ❌ Sin API → Si tienen stack tech complejo
- ❌ Dashboard básico → Si quieren analytics
- ❌ Sin Account Manager → Si necesitan soporte

**Growth → Enterprise (triggers de upgrade):**
- ❌ Límite 80 usuarios → Si >100 personas
- ❌ Sin white-label completo → Si es mandatorio
- ❌ Sin integraciones SAP/Workday → Si las usan

### 3. Upgrades Automáticos

```
TRIGGER: Cuando cliente excede límite del tier

Ejemplo Starter (20 usuarios):
├─ Mes 3: Cargan 22 usuarios
├─ Sistema: "Has excedido límite. Upgrade a Growth?"
├─ Acción: Auto-upgrade + prorate (cobro diferencia)
└─ Resultado: Expansion revenue sin sales touch

📊 Conversión auto-upgrade: 78% (vs 45% manual)
```

### Validación de Límites

```python
def check_plan_limits(company):
    """Verifica si la empresa excede los límites de su plan"""
    limits = PLAN_LIMITS.get(company.plan, PLAN_LIMITS["starter"])
    
    current_users = company.user_set.filter(is_active=True).count()
    current_programs = company.program_set.filter(status="active").count()
    
    exceeded = []
    if current_users >= limits["max_users"]:
        exceeded.append("users")
    if current_programs >= limits["max_programs"]:
        exceeded.append("programs")
    
    return {
        "exceeded": exceeded,
        "suggest_upgrade": len(exceeded) > 0,
        "current": {"users": current_users, "programs": current_programs},
        "limits": limits,
    }
```
